51 /*
52  A Collection of Fortran Array-like Objects
53 
54 
55  The FabArray<FAB> class implements a collection (stored as an array) of
56  Fortran array-like objects. The parameterized type FAB is intended to be
57  any class derived from BaseFab<T>. For example, FAB may be a BaseFab of
58  integers, so we could write:
59 
60  FabArray<BaseFab<int> > int_fabs;
61 
62  Then int_fabs is a FabArray that can hold a collection of BaseFab<int>
63  objects.
64 
65  FabArray is not just a general container class for Fortran arrays. It is
66  intended to hold "grid" data for use in finite difference calculations in
67  which the data is defined on a union of (usually disjoint) rectangular
68  regions embedded in a uniform index space. This region, called the valid
69  region, is represented by a BoxArray. For the purposes of this discussion,
70  the Kth Box in the BoxArray represents the interior region of the Kth grid.
71 
72  Since the intent is to be used with finite difference calculations a
73  FabArray also includes the notion of a boundary region for each grid. The
74  boundary region is specified by the ngrow parameter which tells the FabArray
75  to allocate each FAB to be ngrow cells larger in all directions than the
76  underlying Box. The larger region covered by the union of all the FABs is
77  called the region of definition. The underlying notion is that the valid
78  region contains the grid interior data and the region of definition includes
79  the interior region plus the boundary areas.
80 
81  Operations are available to copy data from the valid regions into these
82  boundary areas where the two overlap. The number of components, that is,
83  the number of values that can be stored in each cell of a FAB, is either
84  given as an argument to the constructor or is inherent in the definition of
85  the underlying FAB. Each FAB in the FabArray will have the same number of
86  components.
87 
88  In summary, a FabArray is an array of FABs. The Kth element contains a FAB
89  that holds the data for the Kth grid, a Box that defines the valid region
90  of the Kth grid.
91 
92  A typical use for a FabArray would be to hold the solution vector or
93  right-hand-side when solving a linear system of equations on a union of
94  rectangular grids. The copy operations would be used to copy data from the
95  valid regions of neighboring grids into the boundary regions after each
96  relaxation step of the iterative method. If a multigrid method is used, a
97  FabArray could be used to hold the data at each level in the multigrid
98  hierarchy.
99 
100  This class is a concrete class not a polymorphic one.
101 
102  This class does NOT provide a copy constructor or assignment operator.
103 */

1119  m_fabs_v.push_back(factory.create(tmpbox, n_comp, fab_info, K));

1121 
1122 #ifdef BL_USE_TEAM
1123  if (shmem.alloc)
1124  {
1125  const int teamlead = ParallelDescriptor::MyTeamLead();
1126

1128  shmem.n_points = 0;

1145  }
